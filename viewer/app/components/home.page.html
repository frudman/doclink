Hi There! This is (yet another?) public place for developers to use or share their code. More specifically, to share web-based components, aka doctype-components. (our wikipedia entry) A web component is simply a single-file containing all the html, css, and javascript required to display and interact with that component of yours. This component can then be used in your apps, or, if you make it public, everyone else's apps! Each component is kept as a SINGLE FILE that looks like a very simplified html page (more here). [click to see a sample] Anyone can add components to this repositary, for others to use. Requirements to create and add a component are pretty simple (see here). And we encourage all to add to our community! (but you can also create a private or corporate account if you'd like, so only apps you approve can use your compoments)

single file components

Can consist of multiple files and assets!!!

but each of those must be referenced from within the primary sfc file

highlights

A doctype component is a single html-like file (in fact, we recommend using a .html extension so various editors can use proper intellisense when editing) a DC can make use of additional assets (i.e. separate files) but these assets must then be explictly referenced within the main DC file A DC is meant to be used AS-IS (downloaded into a browser) without need for BUILD (recommendation is to NOT build for smaller, simple compoments). However, a DC can be build (for a dev's peace of mind!) To quickly and easily package web-ui code/functions/features to be easily shared and consumed by [possible unrelated/other] web apps. No build chain needed (and building, if wished for) can be fully automated outside the dev process (i.e. by the repo itself; mostly for compression purposes: mostly minification & gzipping) To leverage current [excellent] crop of native toolsets for front end devs (e.g. visual studio code) - works really well with html code, css stylesheets, and javascript code.
